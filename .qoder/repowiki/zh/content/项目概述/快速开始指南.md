## 简介

Secbot（原 Hackbot）是一个智能化的自动化渗透测试机器人，具备 AI 驱动的安全测试能力。该项目采用多智能体架构，结合 LLM 推理能力和丰富的安全工具集，为用户提供完整的自动化安全测试解决方案。

## 系统要求

### 最低系统要求
- **Python**: 3.10+
- **包管理器**: uv（快速 Python 包管理器）
- **推理引擎**: Ollama（本地 LLM 推理）
- **数据库**: SQLite（内置，无需额外配置）

### 推荐硬件配置
- **内存**: 8GB RAM（推荐 16GB+）
- **存储**: 至少 10GB 可用空间
- **网络**: 稳定的互联网连接（用于模型下载和在线功能）

### 操作系统支持
- **Windows**: Windows 10/11（推荐）
- **macOS**: macOS 10.15+
- **Linux**: Ubuntu 18.04+/CentOS 7+

## 安装方式选择

### 三种安装方式对比

| 安装方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|----------|
| **发布版** | 无需 Python 环境，即下即用 | 功能受限，无法自定义 | 快速体验，生产部署 |
| **源码安装** | 完整功能，可定制，便于开发 | 需要 Python 环境，安装复杂 | 开发测试，深度使用 |
| **Docker 安装** | 环境隔离，易于部署 | 需要 Docker 知识，资源占用 | 生产环境，容器化部署 |

### 安装方式选择建议

- **新手用户**: 选择发布版，快速体验核心功能
- **开发者**: 选择源码安装，便于调试和二次开发
- **生产环境**: 选择 Docker 安装，确保环境一致性

## 从源码安装

### 步骤 1：克隆仓库

```bash
git clone https://github.com/iammm0/secbot.git
cd secbot
```

### 步骤 2：安装 uv 包管理器

```bash
# Windows/Linux/macOS
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 步骤 3：安装项目依赖

```bash
# 安装依赖（推荐）
uv sync

# 或使用 Makefile
make install
```

### 步骤 4：安装 Ollama 推理引擎

```bash
# Windows
winget install ollama

# macOS/Linux
curl -fsSL https://ollama.ai/install.sh | sh

# 启动 Ollama 服务
ollama serve
```

### 步骤 5：下载必要模型

```bash
# 下载推理模型
ollama pull gemma3:1b

# 下载嵌入模型
ollama pull nomic-embed-text
```

### 验证安装

```bash
# 检查 Ollama 服务状态
ollama list

# 运行基本测试
uv run pytest tests/test_db_connection.py
```

## 发布版安装

### Windows 版本

1. **下载发布包**
   - 访问 [Releases 页面](https://github.com/iammm0/secbot/releases)
   - 下载对应平台的压缩包（如 `secbot-windows-amd64.zip`）

2. **解压并配置**
   ```bash
   # 解压到指定目录
   unzip secbot-windows-amd64.zip
   
   # 配置 API Key（必需）
   echo "DEEPSEEK_API_KEY=sk-xxxxxxxx" > secbot.env
   ```

3. **启动应用**
   ```bash
   cd secbot
   .\secbot.exe
   ```

### macOS/Linux 版本

```bash
# 下载并解压
wget https://github.com/iammm0/secbot/releases/latest/download/secbot-linux-amd64.zip
unzip secbot-linux-amd64.zip

# 设置执行权限
chmod +x secbot

# 配置 API Key
export DEEPSEEK_API_KEY=sk-xxxxxxxx

# 启动应用
./secbot
```

## Docker 安装

### 使用 Docker Compose

```bash
# 克隆仓库
git clone https://github.com/iammm0/secbot.git
cd secbot

# 构建镜像
docker-compose -f docker-compose.prod.yml build

# 启动服务
docker-compose -f docker-compose.prod.yml up -d
```

### 手动构建镜像

```bash
# 构建应用镜像
docker build -t secbot:latest .

# 运行容器
docker run -d \
  --name secbot \
  -p 8000:8000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  secbot:latest
```

### Docker 环境配置

```yaml
# docker-compose.yml 示例
version: '3.8'
services:
  secbot:
    image: secbot:latest
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
      - DATABASE_URL=sqlite:///./data/secbot.db
    depends_on:
      - ollama
    restart: unless-stopped

  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ./ollama_data:/root/.ollama
    restart: unless-stopped
```

## 环境配置

### 配置文件设置

1. **复制环境配置**
   ```bash
   cp .env.example .env
   ```

2. **编辑 .env 文件**
   ```env
   # Ollama 配置
   OLLAMA_BASE_URL=http://localhost:11434
   OLLAMA_MODEL=gemma3:1b
   OLLAMA_EMBEDDING_MODEL=nomic-embed-text
   
   # 数据库配置
   DATABASE_URL=sqlite:///./data/secbot.db
   
   # API 配置
   API_PORT=8000
   API_HOST=0.0.0.0
   
   # 日志配置
   LOG_LEVEL=INFO
   LOG_DIR=./logs
   ```

### 数据库配置

```python
# 数据库配置示例
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

# SQLite 配置
DATABASE_URL = "sqlite:///./data/secbot.db"

# 创建引擎
engine = create_engine(DATABASE_URL, connect_args={"check_same_thread": False})

# 创建会话
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
```

### 模型配置

## 常见问题解决

### 安装问题

#### uv 安装失败
```bash
# 解决方案 1：使用官方安装脚本
curl -LsSf https://astral.sh/uv/install.sh | sh

# 解决方案 2：使用 pip 安装
pip install uv

# 解决方案 3：使用包管理器
# macOS: brew install uv
# Ubuntu: sudo apt install uv
```

#### 依赖安装失败
```bash
# 清理缓存并重新安装
uv cache clean
uv sync --refresh

# 使用国内镜像源
uv pip install --index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### 运行问题

#### 端口占用
```bash
# 检查端口占用
netstat -ano | findstr :8000

# 杀死占用进程
taskkill /PID <进程ID> /F

# 或重启服务
uv run python -m router.main
```

#### 权限问题
```bash
# Linux/macOS 权限设置
chmod +x secbot
chmod 755 secbot

# 数据目录权限
sudo chown -R $(whoami) data/
sudo chmod -R 755 data/
```

### 功能问题

#### Ollama 连接失败
```bash
# 检查 Ollama 服务状态
ollama serve

# 重启 Ollama 服务
ollama kill
ollama serve

# 检查网络连接
ping localhost:11434
```

#### 模型下载失败
```bash
# 检查网络连接
curl -I https://ollama.ai

# 使用代理下载
ollama pull --insecure gemma3:1b

# 清理缓存重新下载
ollama ps -a
ollama rm <模型ID>
ollama pull gemma3:1b
```

## 故障排除指南

### 系统级故障

#### Python 环境问题
```bash

python --version

# 检查包安装
pip list | grep secbot

# 重新安装
pip uninstall secbot
pip install -e .
```

#### 端口冲突诊断
```bash
# Windows
netstat -ano | findstr :8000

# Linux/macOS
lsof -i :8000
# 或
ss -tulpn | grep :8000

# 杀死进程
kill -9 <PID>
```

### 网络连接问题

#### API 连接失败
```bash
# 检查后端服务
curl -I http://localhost:8000

# 检查防火墙设置
# Windows: netsh advfirewall firewall show rule name=all
# Linux: sudo ufw status
# macOS: sudo pfctl -sr

# 检查代理设置
echo $HTTP_PROXY
echo $HTTPS_PROXY
```

#### 数据库连接问题
```bash
# 检查数据库文件
ls -la data/
file data/secbot.db

# 检查权限
ls -la data/secbot.db
chmod 666 data/secbot.db

# 重建数据库
rm data/secbot.db
uv run python -c "from database.manager import DatabaseManager; DatabaseManager()"
```

### 性能优化

```bash
# 监控内存使用
```
